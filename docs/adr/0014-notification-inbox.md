# 0014. 알림함: send_expo_push 단일 지점 기록

- 상태: 채택
- 날짜: 2026-07-09

## 배경

푸시 알림이 발송 후 사라져(fire-and-forget) 앱 안에서 지난 알림을 볼 수 없었다. 놓친 버디 요청·안전 알림을 확인할 인박스가 필요했다.

## 결정

`notifications` 테이블을 만들고, **모든 알림이 지나가는 send_expo_push 함수 안에서** 푸시 발송 전에 행을 삽입한다. 트리거·호출부(버디/참가/경보/채팅)를 하나도 수정하지 않고 전 알림이 자동으로 인박스에 쌓인다. 클라이언트는 Realtime 구독으로 배지를 갱신하고, 알림함을 열면 일괄 읽음 처리한다. 벤치마크: Instagram 알림 탭.

## 대안

- 각 트리거마다 인박스 삽입 추가: 중복 코드, 새 알림마다 누락 위험 — 기각.
- 클라이언트 로컬 저장: 기기 바뀌면 유실, 미수신 푸시는 기록 안 됨 — 기각.

## 결과

- 이후 추가되는 어떤 알림도 send_expo_push만 쓰면 자동으로 인박스에 남는다.
- 감수: 인박스 무한 증가 — 추후 보존기간(예: 90일) 정리 배치 필요.
