# 0014. 알림함: send_expo_push에서 DB 기록

- 상태: 채택
- 날짜: 2026-07-09

## 배경

푸시는 발송되지만 앱 안에 "받은 알림 목록"이 없어, 놓친 알림을 다시 볼 수 없었다.

## 결정

모든 알림이 지나는 send_expo_push() 함수가 발송 직전 notifications 테이블에 한 줄 기록하게 했다. 버디·참가·경보·채팅 등 모든 알림이 코드 한 곳 수정으로 전부 알림함에 쌓인다. 지도 홈/헤더에 🔔 + 안 읽은 배지, 열면 읽음 처리. 실시간(Realtime) 반영.

## 대안

- 각 트리거마다 개별 insert: 중복·누락 위험 — 기각(단일 통로 채택).

## 결과

- 새 알림 종류를 추가해도 자동으로 알림함에 포함. 감수: 읽음 처리는 화면 진입 시 일괄(개별 읽음 아님).
