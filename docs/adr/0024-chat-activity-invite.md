# 0024. 채팅 활동 초대 카드

- 상태: 채택
- 날짜: 2026-07-13

## 맥락

채팅방(버디 1:1 / 크루 / 활동 단체)에서 진행예정·진행중인 아웃도어 활동을
"참여 링크"처럼 공유해, 받은 사람이 눌러 바로 참가신청할 수 있어야 한다.
벤치마크: 카카오 일정 공유·Discord 이벤트·Meetup 링크 프리뷰 — 공통적으로
텍스트 링크가 아니라 **정보가 보이는 카드**(제목·시간·장소·상태) + 탭 시 참가 화면.

## 결정

- **공유 범위**: 내가 호스트인 활동 + 내가 수락된 참가자인 활동, 상태 scheduled/active,
  동행구함 포스트만 (`my_shareable_activities()` RPC).
- **인코딩**: 라이브맵과 같은 특수 메시지 방식 — 본문 `[activity:<postId>]`.
  세 채팅의 기존 메시지 파이프라인을 그대로 쓰므로 새 테이블 불필요.
- **카드 렌더링**: `ActivityInviteCard`가 `post_detail(pid)` RPC로 **현재 상태를 실시간 조회** —
  공유 후 종료되면 카드가 자동으로 "종료" 배지 + 신청 불가로 바뀐다(상태를 본문에 굳히지 않음).
- **거리 무관 열람**: `post_detail` RPC는 거리 제한이 없고, 동행구함 체크인 포스트는
  공개/친구 여부와 무관하게 열람 허용(참여 목적). 포스트 상세 화면은 nearby 피드에 없으면
  이 RPC로 폴백 → 멀리 있는 크루원도 열고 참가신청 가능.
- **UI**: 세 채팅 입력창 왼쪽에 공용 `ActivityShareButton`(＋) → `ActivityShareSheet`
  (내 공유 가능 활동 목록) → 선택 시 카드 전송. 컴포넌트 하나로 3곳 재사용.

## 대안

- 카드에 제목·시간을 본문으로 굳혀 저장(조회 0): 상태 변화(종료)를 못 따라감 → 실시간 조회 채택.
- 딥링크 URL 공유: 앱 외부 공유엔 유용하나 인앱 채팅엔 카드가 더 직관적 → 후순위.
- 별도 invite 테이블: 기존 메시지 파이프라인으로 충분해 과설계 → 기각.
