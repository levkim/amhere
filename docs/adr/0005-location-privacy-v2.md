# 0005. 위치 프라이버시: 서버 가상화 (10km / 500m)

- 상태: 채택
- 날짜: 2026-07-06

## 배경

위치 기반 소셜 앱의 최대 리스크는 스토킹이다. 반면 "주변에 누가 있는지"는 앱의 핵심 가치라 위치를 아예 숨길 수도 없다.

## 결정

실제 좌표는 **절대 서버 밖으로 내보내지 않는다**. `nearby_users` RPC가 서버에서 좌표를 가상화해 반환한다:

- 모든 가입자: 전 세계 공개하되 **반경 10km 내 가상 위치**
- 서로 친구(수락된 버디)이고 20km 이내: **반경 500m 내 가상 위치**
- 고스트 모드: 지도에서 완전 제외

가상 위치는 user_id 기반 시드로 고정되어 지도에서 튀지 않는다 (`fuzz_location`, PostGIS `st_project`). `user_locations` 테이블에는 SELECT 정책이 없어 클라이언트가 원좌표를 읽을 방법이 없다.

## 대안

- **클라이언트에서 좌표 뭉개기**: API 응답에 원좌표가 노출되므로 보안이 아님 — 기각
- **정확 위치 상호 공개 (Zenly식)**: 낯선 사람 매칭이 전제인 이 앱에는 위험 — 기각

## 결과

- 스토킹 방어가 구조적으로 보장됨 (신뢰가 곧 제품 경쟁력)
- 감수: "바로 옆에 있는" 사람도 지도상 최대 10km 떨어져 보일 수 있음 — 만남 조율은 채팅으로 유도
