# ADR (Architecture Decision Records)

이 폴더는 **"왜 그렇게 만들었는가"** 를 기록하는 곳이다.

## ADR이란?

중요한 기술 결정을 내릴 때마다 남기는 **한 장짜리 메모**. 코드에는 "무엇을" 만들었는지만 남고, "왜" 그렇게 했는지는 사라진다. ADR이 그 "왜"를 보존한다.

## 언제 쓰나?

- 기술/라이브러리를 선택했을 때 (예: Mapbox vs 무료 지도)
- 되돌리기 어려운 구조를 정했을 때 (예: 위치 프라이버시 모델)
- 뭔가를 **하지 않기로** 했을 때 (예: MVP에서 결제 제외)

버그 수정이나 사소한 코드 변경은 기록하지 않는다.

## 기록 단위: ADR 1개 = 결정 1개

기능 단위가 아니라 **결정 단위**로 나눈다. 한 기능에 결정이 두 개면 ADR도 두 개다.
판별 질문: *"6개월 뒤 누군가 '왜 이렇게 했어?'라고 물을 만한가?"* — 그렇다면 기록한다.

- 제품 방향(누구에게 뭘) → `docs/PRD.md`
- 현재 구조 스냅샷(지금 어떻게 생겼나) → `docs/ARCHITECTURE.md`
- 개별 기술 결정의 "왜" → **여기 (ADR)**
- 구현 디테일 → 코드 주석

## 작성 규칙

1. 파일명: `NNNN-짧은-제목.md` (번호는 이어서 증가)
2. 아래 템플릿 사용 — **한 페이지를 넘기지 않는다**
3. 결정이 나중에 뒤집히면 지우지 말고 상태를 `대체됨 (→ NNNN)` 으로 바꾼다
4. 아래 목록표에 한 줄 추가한다

## 템플릿

```markdown
# NNNN. 제목 (결정을 한 문장으로)

- 상태: 채택 | 대체됨 (→ NNNN) | 폐기
- 날짜: YYYY-MM-DD

## 배경 (왜 결정이 필요했나)

## 결정 (무엇을 하기로 했나)

## 대안 (무엇을 검토하고 왜 버렸나)

## 결과 (이 결정으로 감수하는 것)
```
