# 0021. 크루 폭파: 멤버 전원 퇴장 후에만 (서버 강제)

- 상태: 채택
- 날짜: 2026-07-12

## 맥락

크루장이 자신이 만든 크루를 없앨 방법이 필요하다. 벤치마크:
- **네이버 밴드**: 멤버를 전원 내보낸 후에만 모임 삭제 가능
- **카카오 오픈채팅**: 방장이 멤버가 있어도 즉시 폭파
- **Discord**: 서버 이름 타이핑으로 강한 확인 후 즉시 삭제

## 결정

**밴드 방식**을 채택한다 — 크루장 외 수락된 멤버가 0명일 때만 폭파 가능.
멤버가 모르는 사이 커뮤니티가 증발하는 사고를 구조적으로 막는다.

- 클라이언트: 크루장 전용 "💥 크루 폭파" 버튼. 멤버가 남아 있으면 안내만 하고 진행 불가.
  두 단계 확인(되돌릴 수 없음 명시) 후 삭제.
- 서버: `guard_crew_disband` BEFORE DELETE 트리거가 같은 조건을 강제(00025).
  예약취소 가드(0019)와 동일한 패턴 — 클라이언트 우회 불가.
- 보완: 유령 멤버로 영원히 폭파 불가한 상황을 막기 위해 크루장 "내보내기" UI 추가
  (RLS 권한은 00016부터 있었고 UI만 없었음).
- 삭제 효과: 멤버십·크루 채팅은 cascade 삭제, 크루로 공유한 포스트는
  `crew_id`만 null로 풀리고 글은 유지(00017의 on delete set null).

## 대안

- **즉시 폭파(카카오 방식)**: 구현은 단순하나 멤버 입장에서 예고 없는 증발 → 기각.
- **soft delete(숨김)**: 복구 가능하지만 상태 복잡도 증가, 초기 단계에 과함 → 기각.
