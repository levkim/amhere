# 0007. 배포: EAS Update(OTA)로 JS 변경분 즉시 배포

- 상태: 채택
- 날짜: 2026-07-06

## 배경

기능 고도화가 반복되면서 매번 EAS 클라우드 빌드(10~20분) + APK 재설치가 필요했다. JS 코드만 바뀌는 경우가 대부분인데도 전체 재빌드를 반복하는 것은 개발 속도를 크게 떨어뜨린다.

## 결정

EAS Update(OTA)를 도입한다. 빌드 프로파일별 채널(development/preview/production)을 두고, JS 변경은 `eas update --channel <채널>`로 배포하면 설치된 앱이 다음 실행 때 자동 수신한다. `runtimeVersion`은 `appVersion` 정책(현재 0.1.0)을 사용한다.

## 대안

- **매번 재빌드**: 확실하지만 반복 비용이 큼 — 기각
- **CodePush(MS)**: 서비스 종료 수순, Expo 생태계와 이질적 — 기각

## 결과

- JS 수정 → 1분 내 전체 기기 배포. 네이티브 모듈 추가 때만 재빌드
- 감수: **네이티브 부품이 바뀌면 OTA로 배포하면 안 됨** (runtimeVersion이 지켜주지만, 앱 버전을 올리고 재빌드하는 규칙을 지켜야 함). 채널·버전 관리 개념이 하나 늘어남
