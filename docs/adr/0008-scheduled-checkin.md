# 0008. 예약 체크인 (미래 시작 시각 + scheduled 상태)

- 상태: 채택
- 날짜: 2026-07-07

## 배경

아웃도어 체크인이 "지금 즉시 출발"만 지원했다. 사용자는 "2시간 후 하이킹 시작"처럼 미리 예약하고, 같이 갈 버디를 모집하고 싶어 했다.

## 결정

체크인에 시작 예약 시각(scheduled_start_at, 최대 7일 뒤)과 새 상태 `scheduled`를 도입했다.

- **예약(scheduled)**: 미래 시작. 아직 안전 감시를 시작하지 않는다. 홈 배너에 빨간 테두리 + 시작까지 실시간 카운트다운.
- **시작 시각 도달**: 로컬 알림 "출발하셨나요?" → 사용자가 확인하면 `active`로 전환(감시 시작). (서버측 30분 무응답 자동 전환은 후속 과제)
- 벤치마크: Uber 예약 승차(카운트다운 카드), Meetup(예정 이벤트로 사람 모집), bSafe/Life360(시작 확인으로 허위 알림 방지).

예약을 "진행 중"으로 취급하지 않는 것이 핵심 — 그러지 않으면 서버 미귀환 감시가 아직 출발도 안 한 사용자에게 헛알림을 보낸다.

## 대안

- **예약 없이 즉시 시작만**: 단순하지만 버디 사전 모집 불가 — 기각
- **예약 시 바로 active 처리**: 미귀환 감시가 오작동해 허위 알림 위험 — 기각

## 결과

- 미리 계획을 세우고 버디를 모집할 수 있음. 앱 재시작해도 배너가 DB에서 복원됨.
- 감수: 앱이 닫힌 상태의 "30분 무응답 자동 감시 시작"은 서버(pg_cron) 작업이 필요 — 다음 단계로 미룸. 현재는 앱이 열려 있거나 로컬 알림에 반응할 때 동작.
