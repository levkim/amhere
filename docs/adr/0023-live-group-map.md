# 0023. 활동 라이브 맵: 멤버 전원 상호 위치 공유

- 상태: 채택
- 날짜: 2026-07-13

## 맥락

진행 중인 아웃도어 활동에서 호스트·동행이 서로의 위치를 실시간으로 봐야
낙오·이탈을 빨리 알아챌 수 있다. 벤치마크: Life360/Zenly(그룹 전원 상호 공유 + 경로 꼬리),
Strava Beacon(일방향 링크), 트랭글(그룹 산행 멤버 지도). **전원 상호 공유(Life360 방식)** 채택
— 낙오자도 앞사람을 볼 수 있어야 안전 가치가 완성된다.

## 결정

- **데이터**: `live_locations` (check_in_id+user_id PK, 좌표, trail jsonb ≤60pt, updated_at).
  RLS `is_activity_member()` — 호스트+수락된 동행만 조회. 쓰기는 본인 행만.
  삭제는 본인 행 + 호스트(활동 종료 정리). Realtime publication 등록.
  pg_cron `live-locations-cleanup` 매시 30분, 12시간 지난 행 삭제(비정상 종료 대비).
- **업로드 경로 2갈래**:
  - 호스트: 기존 TRACK_TASK 콜백에 `pushLive()` 편승 (이중 GPS 태스크 방지).
    체크인 시작 시 `setLiveSession()`, 종료 시 정리.
  - 동행: 전용 `LIVE_TASK` (30m/20s, 트래킹보다 성김 — 배터리). 로컬 트랙은 기록 안 함.
  - 라이브 세션은 AsyncStorage 단일 키 — 백그라운드 태스크가 세션 없으면 업로드 안 함.
- **화면** `activity/[id]/live`: 전체 화면 Mapbox. 멤버별 색상 마커(닉네임 태그)+경로 꼬리,
  5분 무갱신 시 회색(통신 두절 신호). 탭 → 좌표·갱신시각·꼬리 이동거리 카드.
  하단: 멤버 칩, "내 위치 공유" 토글, "채팅에 공유".
- **채팅 공유**: 스냅샷 이미지 대신 특수 메시지 `[livemap]` → 채팅에서 카드로 렌더링,
  탭하면 살아있는 라이브 맵으로 이동. (이미지보다 유용하고 저장 비용 0)
- **프라이버시**: 정확 좌표는 활동 멤버 한정·활동 시간 한정. 공유는 각자 토글로 중단 가능,
  중단 시 서버 행 즉시 삭제. 피드의 반경 처리(fuzzing) 모델과 별개의 동의 기반 공유.

## 대안

- 호스트만 보기(Strava Beacon): 낙오자가 그룹을 못 봄 → 기각 (사용자 결정).
- 채팅에 지도 스크린샷 첨부: 찍는 순간 낡은 정보 + 저장 비용 → 라이브 링크 카드로 대체.
- 별도 위치 소켓 브로드캐스트(supabase broadcast): DB 미저장이라 늦게 합류한 멤버가
  기존 꼬리를 못 봄 → 테이블 저장 방식 채택.
