# 0010. 임시 지킴이 + 참가신청(호스트 승인·단체 채팅)

- 상태: 채택
- 날짜: 2026-07-07

## 배경

체크인이 개인 안전 기능에 머물렀다. (1) 앱 회원인 버디를 이번 활동만의 비상연락(지킴이)으로 쓰고 싶고, (2) '동행구함' 활동에 다른 회원이 참여할 수 있어야 커뮤니티가 커진다.

## 결정

- **임시 지킴이**: check_ins.guardian_ids(uuid[])에 버디 지정. 시작 시 notify_guardians RPC가 지킴이에게 '지정됨' 푸시. (미귀환 경보 발송은 서버 안전감시 작업 때 완성 예정)
- **참가신청(호스트 승인제)**: activity_participants 테이블. 포스트↔체크인 연결(posts.check_in_id)로 '동행구함' 글에서 참가신청 → 호스트 수락/거절 → 알림. 벤치마크: Meetup RSVP.
- **활동별 단체 채팅**: activity_messages 테이블, 멤버(호스트+수락 참가자)만 읽기/쓰기(RLS + is_activity_member). 벤치마크: Strava/Meetup 이벤트 채팅.
- 접근 통제는 전부 RLS(security definer 헬퍼)로 서버에서 강제.

## 대안

- **자동 참가(오픈 조인)**: 낯선 사람이 안전 검증 없이 합류 — 기각(승인제 채택)
- **호스트와 1:1 채팅**: 기존 인프라 재사용은 쉽지만 함께 조율이 어려움 — 단체 채팅 채택

## 결과

- 안전(지킴이·승인제)과 커뮤니티(참가·단체 채팅)를 동시에. 위치 가상화·공개 범위와 함께 다층 안전.
- 감수: 미귀환 시 지킴이 실제 경보는 서버 pg_cron 안전감시(미배포) 완성이 필요 — 후속 과제.
