# 0016. 크루(그룹): 지속 모임 + 크루장 선택제 가입

- 상태: 채택
- 날짜: 2026-07-09

## 배경

버디·참가신청은 단발 활동 중심이라 재방문을 만드는 "소속"이 없었다. PRD v1의 크루 기능이 필요해졌다.

## 결정

크루 = 이름·소개·대표 활동·지역·이모지·크루장. 벤치마크: Strava 클럽, Meetup, 당근 모임, 밴드.

- **가입 방식은 크루장이 선택**: open(바로 가입) / approval(승인제). RLS insert 정책이 join_mode에 따라 status(accepted/pending)를 서버에서 강제.
- 진입점: 버디 탭 상단 [버디|크루] 세그먼트 (탭바는 4개 유지).
- 크루 홈: 소개+멤버(👑 크루장, 승인 대기 관리)+가입/탈퇴+크루 단체 채팅(crew_messages, is_crew_member RLS — 참가신청 채팅 패턴 재사용).
- 탐색: list_crews() RPC가 멤버 수·내 상태 포함, 멤버 수 순 정렬.
- 알림: 가입 신청→크루장, 승인/거절→신청자, 크루 채팅→멤버 전원(발신자 제외). 모두 send_expo_push 경유 → 알림함에도 자동 기록.

## 대안

- 새 하단 탭: 노출은 좋지만 탭바 복잡화 — 세그먼트 채택.
- 모두 승인제/모두 오픈: 크루 성격이 달라 일괄 강제는 부적합 — 선택제 채택.

## 결과

- 지속 모임의 그릇 완성, 기존 인프라(채팅·승인·알림) 재사용으로 전부 OTA 배포.
- 감수: 크루 전용 피드·크루 활동(체크인 연결)은 후속 단계. 크루 채팅 푸시는 전원 발송이라 활성 크루에선 알림이 많을 수 있음(뮤트 기능 후속 검토).
