# 0009. 포스트 공개 범위 (전체 공개 / 친구에게만 / 비공개)

- 상태: 채택
- 날짜: 2026-07-07

## 배경

체크인 소식 공유가 켬/끔(전체 공개/비공개) 뿐이었다. 특히 **예약(미래) 활동**을 낯선 사람에게 공개하면 "언제 어디로 혼자 가는지"가 미리 알려져 스토킹·대기 위험이 실시간 공유보다 더 크다.

## 결정

포스트에 공개 범위 3단계를 도입: `public`(전체) / `friends`(수락된 버디에게만) / 비공개(포스트 자체를 안 만듦).

- 서버: posts.visibility 컬럼 + `are_buddies()` 함수. `nearby_posts` RPC와 RLS SELECT 정책 모두 친구 여부로 필터 → 친구 공개 글은 작성자·친구만 조회 가능(직접 조회 방어 포함).
- 클라이언트: 체크인 화면의 공개 범위 선택. **즉시 시작은 전체 공개, 예약(미래)은 친구에게만이 기본값** (사용자가 직접 고르면 그 값 유지).
- 벤치마크: Instagram 친한친구, Strava 활동 공개 범위, Meetup 비공개 이벤트.

## 대안

- **켬/끔 2단계 유지**: 단순하지만 예약 활동의 스토킹 위험을 못 막음 — 기각
- **시간만 흐릿하게 공개**: 대안으로 검토했으나 대상 세분화(친구에게만)가 더 명확하고 안전 — 채택 안 함

## 결과

- 안전(친구에게만)과 커뮤니티(전체 공개)를 상황별로 선택 가능. 위치 가상화(10km/500m)는 그대로 유지되어 이중 보호.
- 감수: 일반 피드 작성(post/new)은 아직 전체 공개 고정 — 필요 시 후속 확장.
