# 0019. 기록 보관함 + 하이라이트 / 예약 취소 가드

- 상태: 채택
- 날짜: 2026-07-09

## 배경

(1) 포스트가 48시간 후 피드에서 내려가면 개인 기록·추억을 돌아볼 곳이 없었다. (2) 예약 체크인에 동행 신청자가 있는데 호스트가 일방적으로 예약을 취소하면 약속이 깨진다.

## 결정

- **보관함**(profile/archive): 내 포스트+체크인을 월별 사진 그리드로 영구 열람 (인스타 보관함 벤치마크). 데이터 추가 조회 없이 기존 my-posts/check-ins 재사용.
- **하이라이트**: posts.highlighted 플래그. 지정하면 프로필 상단 원형 스트립(인스타 하이라이트식)에 고정되고, **만료 후에도** 공개범위·차단을 존중한 채 타인이 열람 가능(RLS 정책 추가). 단일 포스트 뷰어(post/view/[id])는 피드 캐시와 무관하게 직접 조회.
- **예약 취소 가드**: 예약 체크인은 '지금 시작' 불가(예정 시각 도달 시 '출발 확인'만) + 동행 신청자(pending/accepted)가 있으면 취소 불가 — DB 트리거로 서버 강제(Meetup의 호스트 책임 원칙). 수락 참가자에게 '참가 취소' 버튼 추가(전원 나가면 취소 가능해짐).

## 결과

- 신선한 피드(만료)와 영구 기록(보관함·하이라이트)이 공존. 약속 신뢰가 서버 수준에서 보장.
- 감수: 하이라이트 지정 글은 사실상 영구 공개 — 뷰어에서 해제 가능함을 안내.
