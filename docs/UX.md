# Amhere — 화면별 UX 설계

> 버전 0.1 · 2026-07-06 · 구현 상태: MVP 화면 전부 구현됨 (데모 모드 동작)

## 설계 원칙

1. **3초 규칙**: 앱을 열면 3초 안에 "주변에 누가/무슨 일이" 보여야 한다.
2. **신선도가 곧 신뢰**: 모든 정보에 시간·거리를 붙이고, 24시간이 지나면 지운다.
3. **안전은 마찰 없이**: 안전 기능이 귀찮으면 아무도 안 쓴다. 체크인은 탭 3번 이내.
4. **프라이버시 기본값은 보수적으로**: 위치는 기본 '대략', 정확 위치는 상호 수락 후에만.

## 화면별 설계

### 🗺️ 지도 홈 `(tabs)/index`
| 항목 | 내용 |
|---|---|
| 목표 | 주변 활동 상황을 즉시 파악 |
| 주요 액션 | 지도 탐색, 안전 체크인 시작 |
| UI | 지도(현재 placeholder) + 안전 배너 + "지금 주변에서" 미리보기 3건 |
| 상태 | 위치 거부 → 데모 좌표 폴백 + 안내 문구 / 체크인 중 → 배너가 활동 상태로 전환 |
| 리텐션 | 체크인 활성 시 앱 재방문 동기 (체크아웃해야 함) |

### 📍 피드 `(tabs)/feed`
| 항목 | 내용 |
|---|---|
| 목표 | 내 주변의 살아있는 정보 소비 |
| 주요 액션 | 스크롤, FAB(＋)로 포스트 작성 |
| UI | 신선도 안내 문구 → 포스트 카드 리스트(닉네임·거리·시간·태그) → FAB |
| 상태 | 로딩 스피너 / 에러 "네트워크 확인 + 산악 지역 안내" + 재시도 / 빈 상태 "첫 포스트" CTA |
| 마이크로카피 | "모든 포스트는 24시간 후 사라져요 — 지금의 정보만." |
| 리텐션 | 24h 만료 = 매일 새 콘텐츠, "첫 포스트" 명예 |

### ✍️ 포스트 작성 `post/new` (모달)
| 항목 | 내용 |
|---|---|
| 목표 | 10초 안에 정보 남기기 |
| 주요 액션 | 본문 입력 → "포스트 남기기" |
| UI | 본문(200자) + 활동 칩 + 프리셋 태그 칩(설질좋음/결빙주의/혼잡/크루모집/초보환영/뷰맛집) |
| 상태 | 빈 본문 → 버튼 비활성 / 실패 → Alert + 입력 보존 |
| 마이크로카피 | "이 포스트는 현재 위치에 남고 24시간 후 사라져요" |
| 설계 의도 | 태그를 자유입력이 아닌 프리셋으로 → 정보가 구조화되어 검색/필터 가능 (AllTrails식 검증 태그의 기반) |

### 🤝 버디 탭 `(tabs)/buddies`
| 항목 | 내용 |
|---|---|
| 목표 | 받은/보낸 요청 관리, 매칭 진입점 |
| 주요 액션 | 받은 요청 수락/거절, "버디 찾기" |
| UI | 상단 "주변에서 버디 찾기" CTA → 요청 카드(상대·활동·지역·날짜·상태) |
| 상태 | 받은 pending → 수락/거절 버튼 / accepted → "채팅 열기" / 빈 상태 → 버디 찾기 CTA |
| 리텐션 | 수락 대기 중인 요청 = 재방문 동기 (v1: 푸시) |

### 🔍 버디 찾기 `buddy/find`
| 항목 | 내용 |
|---|---|
| 목표 | 주변 활동자 발견 |
| UI | 유저 카드(닉네임·활동·레벨·대략 거리) + "버디 요청 보내기" |
| 프라이버시 | "정확한 위치는 매칭이 성사된 후에만 공개돼요" 상단 고지 — 신뢰 형성의 핵심 문구 |
| 빈 상태 | "주말 오전에 다시 확인해 보세요" (밀도 낮은 초기에 기대 관리) |

### 💌 버디 요청 `buddy/new` (모달)
| 항목 | 내용 |
|---|---|
| 목표 | 부담 없는 요청 작성 |
| UI | 활동 칩 + 날짜 칩(오늘/내일/모레) + 지역 입력 + 메시지(선택) |
| 마이크로카피 | "자기소개나 계획을 간단히 남기면 성사율이 올라가요" |
| 완료 | Alert "요청 완료" → 버디 탭으로 복귀 |

### 💬 채팅 `chat/[requestId]`
| 항목 | 내용 |
|---|---|
| 목표 | 만남 조율 (매칭 성사 후에만 열림) |
| UI | 말풍선(내 것 파랑/상대 회색) + 하단 입력바 |
| 상태 | 전송 실패 → 입력창에 본문 복구 |
| 실시간 | Supabase Realtime 구독 (연결 시) |

### 🛡️ 안전 체크인 `safety/check-in` (모달)
| 항목 | 내용 |
|---|---|
| 목표 | 탭 3번으로 안전망 활성화 |
| UI | 활동 칩 + 예상 시간 칩(2/4/6/8h) + 시작 버튼 |
| 마이크로카피 | "판정은 서버에서 하므로 산에서 앱이 꺼져도 동작합니다" — 신뢰의 근거 제시 |
| 활동 중 | 지도 홈 배너가 🟢 활동 중 + 체크아웃 버튼으로 전환 |

### 🔑 온보딩 `(auth)/*`
웰컴(가치 제안 한 줄) → 전화번호 → OTP 6자리. 3화면, 30초 목표.
데모 모드일 땐 "데모로 둘러보기" 단일 CTA — 가입 장벽 없이 가치를 먼저 체험.

## 향후 UX 부채 (v1에서 해결)

- 포스트 사진 첨부 (현재 텍스트+태그만)
- 지도 위 핀 인터랙션 (Mapbox 활성화 후)
- 버디 요청 시 상대 프로필 상세 보기
- 푸시 알림 (요청 수신/수락, 체크아웃 리마인더)
- 신고/차단 UI (백엔드는 준비됨)
