# Amhere (여기있어) — PRD

> 버전 0.1 · 2026-07-06 · 스택 확정: React Native + Expo (TypeScript) / Mapbox / Supabase

## 1. 제품 개요

**Amhere(여기있어)**는 아웃도어 활동자(프리라이드 스키/보드, 백컨트리, 하이킹, 트레킹, 러닝)를 위한 **위치 기반 실시간 커뮤니티 앱**이다. 혼자이거나 처음 가는 아웃도어 지역에서 겪는 불안·정보 부족·동행 부재 문제를 "지금, 여기, 나와 같은 사람"을 지도 위에서 보여주는 방식으로 해결한다.

**포지셔닝**: Zenly의 즉각적인 지도 UX + Strava의 챌린지/배지 + AllTrails의 검증된 태그 정보를 결합한 **아웃도어 트라이벌 네트워크**.

## 2. 사용자 문제

1. **고립감/안전 불안**: 혼자 백컨트리·산행 시 사고가 나면 아무도 모른다.
2. **정보 비대칭**: 처음 가는 지역의 실시간 상태(설질, 트레일 상태, 혼잡도)를 알 수 없다. 기존 정보는 오래됐거나 검증이 안 된다.
3. **동행 찾기 어려움**: 같은 시간, 같은 장소에서 같은 활동을 하는 사람을 만날 채널이 없다.

## 3. 목표 / 비목표

### 목표 (MVP)
- 지도에서 주변의 활동 중인 사용자·실시간 피드를 3초 안에 보여준다.
- 같은 지역·같은 활동의 버디를 매칭한다.
- 활동 시작/종료 시 지정한 사람에게 안전 알림(체크인/체크아웃)을 보낸다.

### 비목표 (MVP에서 제외)
- 유료 가이드 투어 마켓플레이스 (v2)
- 정밀 GPS 트랙 기록/분석 (Strava 영역, v1 이후)
- 오프라인 지도 전체 다운로드 (v1)
- 웹 버전

## 5. 핵심 기능

### MVP
1. **라이브 맵**: 앱 열면 바로 지도. 주변 활동자(익명/닉네임 선택), 실시간 피드 핀(사진+태그+한줄).
2. **실시간 피드**: 위치에 귀속된 짧은 포스트. "지금 여기 설질 최고 🎿" — 24시간 후 자동 만료(신선도 보장).
3. **버디 매칭**: 활동/지역/날짜/레벨 필터 → 매칭 요청 → 1:1 채팅.
4. **안전 체크인**: 활동 시작 시 예상 종료 시간 설정 → 미체크아웃 시 비상연락처에 마지막 위치 자동 알림.
5. **프로필/인증**: 활동 이력 기반 레벨 뱃지, 전화번호 인증.

### v1
- 크루(그룹) 기능, 활동 기록 요약, 지역별 명예의 전당(게이미피케이션), 오프라인 지도(핵심 지역).

### v2
- 가이드 투어 마켓플레이스(수익화), 장비 거래, 프리미엄 구독(고급 안전 기능·무제한 오프라인 지도).

## 6. 요구사항 요약

- **위치**: 포그라운드 실시간 공유(활동 중), 백그라운드 추적은 안전 체크인 활성 시에만. 정확도·배터리 트레이드오프는 활동 모드별 프리셋.
- **프라이버시**: 위치 공개 3단계(정확/대략 500m/비공개). 기본값은 "대략". 고스트 모드 필수.
- **실시간성**: 피드·위치 갱신 ≤ 5초 (Supabase Realtime).
- **오프라인 내성**: 네트워크 끊겨도 피드 캐시 열람, 포스트는 큐잉 후 재전송. 안전 체크아웃 실패 시 SMS 폴백(v1).

## 7. 엣지 케이스

- 산악 지역 무신호 상태에서 체크아웃 시간 초과 → 서버 기준으로 판정하고, 마지막 수신 위치 기준 알림 발송.
- GPS 드리프트로 인한 위치 오표시 → 속도/고도 기반 이상치 필터.
- 위치 공유 악용(스토킹) → 정확 위치는 상호 수락한 버디에게만, 신고/차단 즉시 반영.
- 스키장 리프트 이동 중 고속 위치 변화 → 활동 모드별 위치 스무딩.

## 8. 기술 고려사항

- **스택**: React Native + Expo(TypeScript), Expo Router, Mapbox(@rnmapbox/maps), Supabase(Auth/PostGIS/Realtime/Storage), EAS Build(Windows에서 iOS 클라우드 빌드).
- **최대 리스크**: 백그라운드 위치 추적의 배터리 소모 및 iOS 백그라운드 제한 → 안전 체크인 활성 시에만 significant-change 기반 저빈도 추적.
- **앱스토어 심사**: 백그라운드 위치 권한은 명확한 사용자 가치 설명 필수(안전 기능이 근거). 위치 데이터 프라이버시 라벨 정확히 기재.

## 10. 런치 범위 & 로드맵

- **파일럿 지역 1곳으로 시작** (예: 겨울 시즌 주요 스키장 1곳 또는 수도권 인기 산 1곳). 위치 기반 앱은 밀도가 생명 — 전국 동시 오픈 금지.
- **로드맵**: M1–2 MVP 개발 → M3 클로즈드 베타(파일럿 지역 크루 30–50명) → M4 오픈 베타 + 스토어 출시 → M5+ v1 기능, 지역 확장.
